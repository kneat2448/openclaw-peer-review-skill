# Telegram Peer Review MVP — Setup, Migration, and GitHub

## 1. Files to copy

From the source workspace, copy these into the target workspace or repo root:

```text
src/
scripts/
dashboard/
data/team_members.json
data/company_profile.example.json
ecosystem.config.cjs
package.json
.env.example
.gitignore
README.md
```

Do **not** copy:
- `package-lock.json` — regenerated by `npm install` on the target
- `data/peer_review.db*` — for a clean deployment; copy only if preserving existing history
- `data/company_profile.json` — local onboarding output; recreate with `setup company` unless preserving the same org setup
- `node_modules/` — always regenerated
- `.env` — never commit; configure fresh on each instance

For history preservation, also copy:
```text
data/peer_review.db
data/peer_review.db-wal
data/peer_review.db-shm
dashboard/public/data/
```

## 3. Configure .env

```env
TELEGRAM_BOT_TOKEN=your_bot_token_from_botfather
TECH_LEAD_USER_ID=your_telegram_user_id
PORT=3000
BASE_URL=https://your-public-url
DB_PATH=./data/peer_review.db
APP_TIME_ZONE=Asia/Kolkata
LATER_REMIND_MS=14400000
```

- `TELEGRAM_BOT_TOKEN`: create a bot with BotFather, copy the token.
- `TECH_LEAD_USER_ID`: your numeric Telegram ID (use @userinfobot to find it).
- `BASE_URL`: public URL for dashboard links — use localtunnel or ngrok locally.
- `LATER_REMIND_MS`: how long after "later" before a reminder is sent (default 4 hours).

## 4. Add team Telegram IDs

The bot can create the local roster for you. Start the bot, send `/start` as the tech lead, and complete onboarding in one message. To rerun it later, send:

```text
setup company
```

Onboarding writes `data/company_profile.json` and rewrites `data/team_members.json`.

Manual edit format for `data/team_members.json`:

```json
[
  {
    "name": "Nitai",
    "role": "Intern",
    "experience_years": 0,
    "telegram_user_id": "8682856829"
  },
  {
    "name": "Deepak",
    "role": "SDE 1",
    "experience_years": 2,
    "telegram_user_id": "123456789"
  }
]
```

Each member must open the bot and send `/start` at least once before the bot can DM them.
Every member must have a numeric `telegram_user_id`. The bot only responds to the configured tech lead and saved team-member Telegram IDs.

---

## 5. Telegram bot strategy

**For a new/clean deployment:**
- Create a new bot with BotFather.
- Use its token in `.env`.

**Reusing the same bot token across instances:**
- Stop the old instance first.
- Telegram long polling only supports one active poller per token.
- Two pollers on the same token will cause missed messages.

## 7. Start the app

**Simple (dies when terminal closes):**

```bash
npm start
```

**Persistent with pm2 (recommended):**

```bash
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup   # follow the printed command to auto-start on reboot
```

Useful pm2 commands:

```bash
pm2 logs peer-review-bot
pm2 restart peer-review-bot
pm2 stop peer-review-bot
pm2 delete peer-review-bot
```

OpenClaw scheduling note: use a persistent process for this skill. `src/scheduler.js` loads review jobs from SQLite, sets timers for future runs, and starts a once-per-minute cron reconciliation for missed due jobs. `pm2 start ecosystem.config.cjs` is the recommended way for the OpenClaw agent to keep scheduled review timing alive.

Expected startup logs:

```
Telegram bot polling started.
Dashboard server listening on http://localhost:3000
```

Health check:

```bash
curl -fsS http://localhost:3000/
# → Peer Review MVP running. Open /dashboard/<project_id>.
```

## 8. GitHub repo setup

To commit the app as a GitHub repo:

```bash
cd /path/to/workspace
git init
git add src/ scripts/ dashboard/ data/team_members.json ecosystem.config.cjs \
        package.json .env.example .gitignore README.md \
        data/company_profile.example.json \
        skills/peer-review-telegram/
git commit -m "feat: peer review telegram MVP"
git remote add origin https://github.com/your-org/peer-review-telegram.git
git push -u origin main
```

**Never commit:**
- `.env`
- `data/company_profile.json`
- `data/peer_review.db*`
- `node_modules/`
- `package-lock.json` (optional — some teams commit it; omit for cross-machine compatibility)

Confirm `.gitignore` includes:

```
.env
node_modules/
data/*.db
data/*.db-wal
data/*.db-shm
data/company_profile.json
logs/
```

## 11. Troubleshooting

### Bot does not respond

```bash
ps -ef | grep 'node src/server.js' | grep -v grep
node scripts/check-env.js
node --input-type=module - <<'NODE'
import dotenv from 'dotenv'; dotenv.config({ override: true });
const r = await fetch(`https://api.telegram.org/bot${process.env.TELEGRAM_BOT_TOKEN}/getMe`);
console.log(r.status, await r.text());
NODE
```

Check:
- One poller per token.
- Sender Telegram ID equals `TECH_LEAD_USER_ID`.
- Restart the app.

### Team members not receiving reviews

- Add numeric `telegram_user_id` values to every member in `data/team_members.json` or rerun `setup company`.
- Ask each user to open the bot and send `/start` once.
- Check `review_send_log` table for error messages:

```bash
node --input-type=module - <<'NODE'
import dotenv from 'dotenv'; dotenv.config({ override: true });
import Database from 'better-sqlite3';
const db = new Database(process.env.DB_PATH || './data/peer_review.db');
console.table(db.prepare('SELECT * FROM review_send_log ORDER BY sent_at DESC LIMIT 20').all());
NODE
```

### Reviewer lost their progress after restart

Sessions are now stored in `reviewer_sessions` DB table — restarts are safe. If a reviewer still can't resume, check:

```bash
node --input-type=module - <<'NODE'
import dotenv from 'dotenv'; dotenv.config({ override: true });
import Database from 'better-sqlite3';
const db = new Database(process.env.DB_PATH || './data/peer_review.db');
console.table(db.prepare('SELECT reviewer_user_id, project_id, step, updated_at FROM reviewer_sessions').all());
NODE
```

### Wrong scheduled time

```env
APP_TIME_ZONE=Asia/Kolkata
```

Restart after changing.

### DB size / WAL growing

```bash
node --input-type=module - <<'NODE'
import dotenv from 'dotenv'; dotenv.config({ override: true });
import Database from 'better-sqlite3';
const db = new Database(process.env.DB_PATH || './data/peer_review.db');
db.pragma('wal_checkpoint(TRUNCATE)');
db.close();
console.log('WAL checkpointed.');
NODE
```

Full reset (deletes all data):

```bash
rm data/peer_review.db data/peer_review.db-wal data/peer_review.db-shm
npm start
```
